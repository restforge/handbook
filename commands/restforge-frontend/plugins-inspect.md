# `npx restforge-designer plugins inspect`

> Tampilkan metadata lengkap satu plugin (file yang di-scaffold, dependencies, kapabilitas).

## Sintaks

```bash
npx restforge-designer plugins inspect --plugin <PLUGIN> [OPTIONS]
```

## Flag

| Flag | Tipe | Wajib | Keterangan |
|------|------|:-----:|-----------|
| `--plugin <PLUGIN>` | string | ✓ | Plugin ID yang akan di-inspect (mis. `vanilla-js-basic`) |
| `--plugins-dir <PATH>` | string | ✗ | Override folder plugins (env: `RESTFORGE_DESIGNER_PLUGINS_DIR`) |

### Inspect Plugin Auth

```bash
npx restforge-designer plugins inspect --plugin=vanilla-js-auth
```

Output akan menambahkan bagian konfigurasi auth (mis. JWT settings, idle timeout, login page).

### Inspect Plugin Custom

```bash
npx restforge-designer plugins inspect --plugin=my-custom-plugin --plugins-dir=./my-plugins
```

## Behavior

Verb `plugins inspect`:

1. **Resolve plugins directory** (sama dengan `plugins list`)
2. **Cari plugin** berdasarkan ID
3. **Muat metadata** dari `plugin.json` (atau setara) dan struktur folder
4. **Tampilkan rincian** ke console

## Detail yang Ditampilkan

Output bervariasi per plugin. Field umum:

| Field | Sumber | Arti |
|-------|--------|------|
| Version | Metadata plugin | Versi plugin |
| Author | Metadata plugin | Pengembang |
| Description | Metadata plugin | Deskripsi singkat |
| Auth required | Metadata plugin | Apakah plugin butuh konfigurasi auth |
| Capabilities | Metadata plugin | Fitur UDF yang didukung plugin |
| Field types | Metadata plugin | Tipe field yang valid di plugin |
| Page types | Metadata plugin | Tipe page yang valid |
| Files scaffolded | Konvensi `init` plugin | File yang dibuat saat `init --plugin=<id>` |
| Files generated | Konvensi `generate` plugin | File yang di-generate per app dan per page |
| External assets | Metadata plugin | Library eksternal (CDN atau bundled) |

## Exit Code

| Exit Code | Kondisi |
|-----------|---------|
| `0` | Inspect sukses |
| `1` | Plugin tidak ditemukan, plugins directory tidak ada, atau metadata invalid |

## Use Case

| Skenario | Behavior |
|----------|----------|
| Mau tahu fitur yang didukung plugin sebelum pilih | `plugins inspect --plugin=<id>` |
| Debug "kenapa field type X di-reject" | Cek `Field types supported` di output |
| Develop plugin custom, validasi metadata | `plugins inspect --plugin=<custom-id> --plugins-dir=./my-plugins` |
| Bandingkan kapabilitas antar plugin | Inspect tiap plugin, lihat section Capabilities |
